# Butter UI Rules

For UI/frontend/design work, use the Butter skill in `skills/`.

Make the touched UI visibly better: stronger visual hierarchy, compact premium surfaces, animated labels/numbers, and tactile motion for state changes.

Load root plus `visual`, `motion`, and `patterns`; load `construction` when coding.

Never use generic shadcn defaults, marketing heroes, nested cards, decorative blobs, generic gradients, thin text, or explanatory in-app copy.

Final check: compact object, neutral shell, stateful accent, `24-40px` expressive radius, `44-56px` controls, tabular numerals, animated text/numbers, Butter springs, linear progress.
