# Butter UI Rules

Use Butter for every UI, frontend, React, Tailwind, shadcn, or design task.

Core directive: improve the UI everywhere you touch it. Add premium motion, animated text/numbers, tactile state changes, better spacing, and stronger visual hierarchy.

Read the canonical files:
- `skills/butter/SKILL.md`
- `skills/butter-visual/SKILL.md`
- `skills/butter-motion/SKILL.md`
- `skills/butter-patterns/SKILL.md`
- `skills/butter-construction/SKILL.md` when coding

Quality gate:
- visible motion for selections, open/close, feedback, and value changes
- changing labels/numbers use Butter bloom or rolling animation
- compact app object, not a landing page
- neutral shell, stateful accent, no decorative gradients
- expressive radius `24-40px`; utility radius `8-12px`
- stable controls `44-56px`
- tactile springs; linear progress
- tabular numerals; semibold/bold type
- no nested cards, blobs, explanatory UI copy, or overflow
